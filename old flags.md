### 2. Flag 权值位域拆解对比

两者的位域重合度很高，但旧版多了一个 0x40：

| 权值 (Hex)                              | 旧版 (2013) 含义                                         | 新版 (2017) 含义                             | 差异说明                         |
| --------------------------------------- | -------------------------------------------------------- | -------------------------------------------- | -------------------------------- |
| 0x80                                    | 基准位 (必须有)                                          | 基准位 (必须有)                              | 完全一致                         |
| <span style="color:red">**0x40**</span> | <span style="color:red">**未初始化 (Unassigned)**</span> | <span style="color:gray">**[已废弃]**</span> | **核心区别：新版不再使用此位段** |
| 0x20                                    | 指针 (Pointer)                                           | 指针 (Pointer)                               | 均未使用过，但底层预留           |
| 0x08                                    | 已赋值 (Initialized)                                     | 已赋值 (Initialized)                         | 完全一致                         |
| 0x04                                    | 数组 (Array)                                             | 数组 (Array)                                 | 完全一致                         |
| 0x02                                    | 整型 (Int)                                               | 整型 (Int)                                   | 完全一致                         |
| 0x01                                    | 字符串 (String)                                          | 字符串 (String)                              | 完全一致                         |

### 3. 常见变量状态的 Flag 表现对照表

将上述位域组合后，我们在反编译 16 进制文件时，看到的具体 Flag 字节差异如下：

| 变量状态                                | 构成逻辑对比  | 旧版 Flag (Hex) | 新版 Flag (Hex)      |
| --------------------------------------- | ------------- | --------------- | -------------------- |
| **未赋值的 Int**<br>int a;              | **旧：**`0x80 | 0x40            | 0x02<br>**新：**0x80 |
| **未赋值的 String**<br>string s;        | **旧：**`0x80 | 0x40            | 0x01<br>**新：**0x80 |
| **未赋值的 Int 数组**<br>int arr[3];    | **旧：**`0x80 | 0x40            | 0x04                 |
| **未赋值的 Str 数组**<br>string arr[3]; | **旧：**`0x80 | 0x40            | 0x04                 |
| **已赋值的 Int**<br>int a = 1;          | **旧：**`0x80 | 0x08            | 0x02<br>**新：**0x80 |
| **已赋值的 String**<br>string s="a";    | **旧：**`0x80 | 0x08            | 0x01<br>**新：**0x80 |
| **已赋值的 Int 数组**<br>`int a         |               |                 |                      |

根据上面的权值组合，旧版编译器在生成 LGD 文件时，字面值表中会写入以下常见 Flag：

#### 基础未赋值变量 (带有 `0x40` 标志)

在旧版中，只要没有赋初值，就会强制打上 `0x80 | 0x40` 的基础烙印：

- **未赋值 int (`0xC2`)**：`0x80 | 0x40 | 0x02`
- **未赋值 string (`0xC1`)**：`0x80 | 0x40 | 0x01`
- **未赋值 int[] (`0xC6`)**：`0x80 | 0x40 | 0x04 | 0x02`
- **未赋值 string[] (`0xC5`)**：`0x80 | 0x40 | 0x04 | 0x01`

#### 基础已赋值变量 (不带 `0x40`，带有 `0x08`)

一旦赋值，旧版的逻辑与新版完全一致，打上 `0x80 | 0x08` 的基础烙印：

- **已赋值 int (`0x8A`)**：`0x80 | 0x08 | 0x02`
- **已赋值 string (`0x89`)**：`0x80 | 0x08 | 0x01`
- **已赋值 int[] (`0x8E`)**：`0x80 | 0x08 | 0x04 | 0x02`
- **已赋值 string[] (`0x8D`)**：`0x80 | 0x08 | 0x04 | 0x01`

#### 罕见指针类变量 (带有 `0x20` 标志)

虽然官方脚本未实际使用过指针，但旧版编译器完整保留了指针的解析逻辑：

- **未赋值 int\* (`0xE2`)**：`0x80 | 0x40 | 0x20 | 0x02`
- **未赋值 string\* (`0xE1`)**：`0x80 | 0x40 | 0x20 | 0x01`
- **已赋值 int\* (`0xAA`)**：`0x80 | 0x20 | 0x08 | 0x02`
- **已赋值 string\* (`0xA9`)**：`0x80 | 0x20 | 0x08 | 0x01`
